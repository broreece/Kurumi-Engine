KURUMI ENGINE – BUILD CHANGE DOCUMENT
Build: Pursuit
Status: Finished

---------------------------------------------------------------------

PURPOSE

This document outlines the architectural and system-level changes made since the 'Quench' release.

'Pursuit' focused on adding core gameplay features involving enemy formations, battles and persistance:

- Created a tiled map format convertor into our .json format. The repo is hosted on git.
- Enemy formations now display on the map, directly interact with the map's visibility and navigability.
- Battles can now start, end and result in world changes.

---------------------------------------------------------------------

RENDERING

- A new submission index has been created to allow tiles to not be rendered out of order. This affects all renderers
    now.

---------------------------------------------------------------------

FORMATIONS AND ACTOR CONTROLLERS

- Smart and dumb tracking controllers now have a fixed range of where they can track their targets.
- Formations now have their own controllers similar to actors. They execute movements based on their current AI.
    They also directly interact with the map state by affecting visibilty and navigability.
- Formations when dead will no longer interact with the world, they will become passable, see-through and remain
    still.

---------------------------------------------------------------------

BATTLE

- Battles can now end when an enemy formation is defeated.
- When battles end; on win or on lose scripts execute allowing for more script interactions such as flags being set
    etc.
- Battles can now start if a formation encounters the party.
- Attacks can now target randomly, all party members, all enemies or everyone.

---------------------------------------------------------------------

TOOLING

- Created a new tool that converts the "Tiled" map format into our map format. Repo at:
    https://github.com/broreece/Kurumi-Tools

---------------------------------------------------------------------

CODE

- Did a minor refractor where in file headers we specified the imports using a fixed comment structure.
- Broke up the map state functions to better fit SOLID principles and improve modulality.

---------------------------------------------------------------------

KEY TAKEAWAYS

- After all the changes to the workflow making new additions have become significantly easier.
- We accumulated a little but more tech debt this build, After this build we'll make a new build resolving these issues
    before we continue development.
- Some of the tasks I had originally penned for a refinement release I moved to pursuit. Mainly the ones involving 
    formations, encountering or reconstruction.
- Whilst the change log is small the result of this build is huge, it effectively allows real gameplay involving
    enemy formations chasing the player, battle now working fully.
